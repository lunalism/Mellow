# Mellow — 제품 요구사항 문서 (PRD) v0.2

> **한 줄 정의.** Mellow는 일상의 따뜻한 순간을 *필름 감성*으로 기록하는 미니 브이로그 & 필름 카메라 앱이다.
> **무드.** 한낮의 햇살 · 카페 BGM · 아늑한 힐링. 따뜻하고 살짝 빛바랜, 사적인 '필름 다이어리'.
> **플랫폼.** iOS 우선 (네이티브 Swift). · **상태.** Phase 0 완료(무드보드·디자인 시스템 v0.1).

### 확정 결정 (v0.2)
- 필터 8종 이름 확정: Sunday(기본)·Honey·Breeze·Hush·Olive·Linen·Ember·Velvet.
- 2단 네이밍 전략 도입 (코어=짧은 단어 / 시즌=시적 구절).
- 수익화: 무료 핵심 카메라 + **일회성 라이프타임 언락** (구독 지양).
- 성공 지표 북극성: **D7 리텐션** (구체 목표치는 베타 후).
- 촬영 비율에 **2:1(시네마틱) v1 포함**.
- 날짜 스탬프: 메타데이터는 항상 저장, 화면 스탬프 **기본 OFF**.
- 다크모드: **v1 라이트 고정**, 다크 토큰은 v1.1.

## 1. 비전 & 포지셔닝

Mellow는 '잘 찍기 위한 도구'가 아니라 '따뜻한 순간을 간직하기 위한 다이어리'다. 사진이든 짧은 영상이든, 같은 필름 룩으로 묶여 하나의 아늑한 세계관을 만든다.

경쟁 지형에서 우리 자리는 **'따뜻한 페이드 + 미니멀 + 영상까지 품는 다이어리'** 라는 빈틈이다.
- 가장 가까운 Dazz는 '카메라 수십 개 + 사진 우선'의 맥시멀리스트.
- 미니멀의 챔피언 구닥은 색이 차갑다.
- 따뜻한 영상 도구(VLLO·Filmm)는 풀 편집기라 '아늑한 카메라'가 아니다.

→ Mellow의 차별점은 *덜어내기*에서 나온다: 구닥의 절제 + Dazz의 따뜻한 영상 필터 + VSCO의 통일된 톤앤매너, 그 교집합.

---

## 2. 목표 (Goals) & 비목표 (Non-goals)

### 목표
- 사진과 영상이 **하나의 따뜻한 필름 룩**으로 통일된다.
- 촬영 → 필터 → 기록까지 **끊김 없이**, 후보정 단계 없이 완성된다 (라이브 프리뷰).
- **타이트하게 큐레이션된 소수 필터**로 '통일된 세계관'을 만든다 (품질 > 개수).
- UI는 **아늑하고 미니멀**하다 — 무거운 기능보다 분위기.

### 비목표 (v1에서 하지 않는 것)
- ❌ 타임라인 영상 편집기 / 풀 편집 스위트 (CapCut류)
- ❌ 셀카·뷰티 보정 중심 기능
- ❌ Android (테스트 기기 부재 — 향후, 크로스플랫폼 재작성 없이 보류)
- ❌ AI 하늘 교체 (피크닉식) — '미래 차별화 카드'로 백로그
- ❌ 필터 수십 개 양산
- ❌ 다크모드 (v1.1로)

## 3. 타깃 사용자

- **핵심:** 20–30대, 일상·여행·카페를 기록하고 싶은 캐주얼 기록자.
- 필름 감성은 좋아하지만 실물 필름 카메라의 번거로움·비용은 싫은 사람.
- '잘 찍는 것'보다 '예쁘게 간직하고 가볍게 공유하는 것'이 목적.
- (전문 크리에이터·편집 헤비유저는 타깃 아님 — 그들은 별도 제품.)

**대표 페르소나 — 지우(27).** 주말 카페·여행을 기록한다. 인스타에 올릴 때 사진끼리 톤이 안 맞는 게 늘 불만. 필름 카메라를 사봤지만 현상이 귀찮아 서랍에 넣어뒀다. "한 번에, 같은 감성으로" 찍고 싶다.

---

## 4. 핵심 가치 & 차별점

1. **통일감.** 사진·영상이 같은 필름 룩으로 묶인다 — 따로 노는 두 앱이 못 주는 가치.
2. **큐레이션.** 8종의 엄선된 필름. 적은 게 강점.
3. **몰입.** 라이브 프리뷰 + 후보정 생략 → '순간에 머무는' 경험.
4. **아늑함.** 따뜻한 페이드 정체성을 끝까지 지킨다.

## 6. 기능 요구사항

### 6.1 카메라 코어
- 실시간 **라이브 필터 프리뷰** (찍기 전에 결과를 본다 = WYSIWYG).
- **사진 / 영상** 모드 전환.
- 초점거리 프리셋, 노출 슬라이더(+자동), 전/후면 전환, 플래시.
- 비율 선택: 9:16 / 4:3 / 1:1 / 2:1.
- 시뮬레이터에 카메라가 없으므로 **실기기 테스트 필수**.

### 6.2 필터 엔진 (앱의 심장)
- CoreImage/Metal 기반, 사진·영상 **공유 엔진**.
- **비파괴(non-destructive)**: 원본 보존 + 선택한 필터 ID를 메타로 저장 → 익스포트 시 적용.
- 필터 **강도 조절** 슬라이더.
- 공통 '재료': 따뜻한 색감, 필름 그레인(영상은 프레임별로 흔들어 움직이는 그레인), 라이트릭/헐레이션, **들린 블랙(faded black)**, 비네팅, (옵션) 날짜 스탬프.

### 6.3 영상 다이어리
- 짧은 클립을 촬영해 **단순 연속 연결**(타임라인 편집기 ❌).
- 붙인 전체 클립에 **하나의 통일된 필름 룩**을 익스포트 시 일괄 적용.
- BGM 1트랙 + 익스포트. **오디오 트랙 보존 필수**.
- 구현: `AVMutableComposition`(잇기) + `AVVideoComposition`(프레임별 `CIFilter`) + `AVAssetExportSession`.

### 6.4 갤러리 / 보관함
- Mellow 전용 보관함, 사진·영상 통합.
- 비파괴 구조이므로 **나중에 필터 교체 가능**.

### 6.5 저장 · 공유
- 고화질 익스포트(프리뷰 색 = 익스포트 색 보장).
- SNS 공유.
- **날짜:** 촬영 날짜 메타데이터는 항상 저장(다이어리). 화면에 찍히는 비주얼 스탬프는 **기본 OFF + 토글**(켜면 세리프 톤).

## 9. 기술 아키텍처

- **스택:** iOS 네이티브 — SwiftUI + AVFoundation + CoreImage/Metal. (Vision/Core ML은 향후 하늘 기능용.)
- **라이브 프리뷰:** `AVCaptureVideoDataOutput` → `CMSampleBuffer` → `CIImage` → 필터 → Metal 뷰(`MTKView`) 렌더 + `AVAssetWriter` 기록.
- **비파괴 영상 익스포트:** 원본 클립 + 필터 ID 저장 → `AVMutableComposition`으로 잇고 → `AVVideoComposition`으로 프레임별 `CIFilter` 적용 → 익스포트.
- **성능 가드레일:**
  - GPU 기반 `CIContext`(Metal) 필수, CPU 금지.
  - 영상 그레인은 프레임마다 시드 흔들기.
  - 실기기(최저 타깃 아이폰) 테스트 — 프레임 드랍·발열 체크.
  - 프리뷰 파이프라인 = 익스포트 파이프라인 (색 어긋남 방지).
  - 오디오 트랙 보존.

## 10. 수익화 (확정)

- **모델:** 무료 핵심 카메라 + **일회성 라이프타임 언락**.
  - 무료: 핵심 카메라, 사진·영상 촬영·저장, 히어로 필터 일부.
  - 라이프타임: 전체 8종 필름 + 고급 질감(그레인·라이트릭 강도 등).
  - 시즌 한정 필름: 추후 별도 판매(소액).
- **원칙 (반면교사):** FIMO·레트리카가 구독·페이월로 별점 테러를 받았다 → **핵심 카메라·저장은 절대 페이월로 막지 않는다. 무료/유료 필터를 화면에서 명확히 구분한다.**

## 13. 리스크 & 남은 오픈 퀘스천

### 리스크
- **스코프 크리프** — '미니멀 영상'을 끝까지 지킬 것. 타임라인 편집기 유혹 차단.
- **필터 IP** — LUT는 반드시 우리가 직접 제작.
- **성능/발열** — 실시간 영상 필터의 GPU 부담.
- **상표** — 'Mellow'는 일반어 → 컴파운드 리스팅(`Mellow — 필름 다이어리`)·로고 식별력 강화. 출시 전 KIPRIS·USPTO 클리어런스.

### 남은 오픈 퀘스천
1. 라이프타임 **가격** 책정.
2. **폰트** 최종 확정(세리프·산세리프).
3. 시즌 필름 첫 라인업(시적 네이밍).
4. 무료/라이프타임 필터 **분배**(무료에 히어로 몇 종?).
