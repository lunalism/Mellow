# Mellow — 화면 플로우 맵 (IA) v0.1

> **전제.** 회원가입·로그인·계정 없음. 완전 **로컬 앱**. 첫 실행 시 권한 프라이밍만 있고, 나머지는 전부 콘텐츠 화면이다.
> **구조.** 카메라를 중심으로 한 **허브-앤-스포크**. 연결 문서: PRD v0.2 · 디자인 시스템 v0.2 · Phase 1 스펙 v0.2.

---

## 1. 내비게이션 모델

- **카메라 = 홈 허브.** 앱을 켜면 (첫 실행 권한 프라이밍 이후) 항상 카메라로 진입.
- 보관함·설정은 카메라에서 한 번에 도달하는 **스포크**. 뒤로 = 카메라 복귀.
- 뷰어는 보관함에서 **푸시**.
- 필터 언락·공유는 **모달 시트**(어디서든 떠서 닫히면 제자리).
- **탭바 없음** (덜어내기) — 카메라가 중심이라 제스처·썸네일·아이콘으로 이동.

```
앱 실행
  └─(첫 실행만)─ 권한 프라이밍 ─┐
                              ▼
  (재실행 시 바로)──────────► 카메라 · 홈 ◄─── (뒤로)
                              ├──► 보관함 ──► 뷰어 ──► 공유 시트(모달)
                              ├──► 설정
                              └──► 필터 언락(모달)
```

## 2. 화면 인벤토리

| 화면 | 유형 | 진입 | 핵심 역할 |
| --- | --- | --- | --- |
| 권한 프라이밍 | 시스템/온보딩 | 첫 실행 | 카메라·마이크·사진 권한 사유 설명 후 요청 |
| **카메라 · 홈** | 화면(허브) | 실행 직후 | 촬영·필터·기록의 중심 |
| 보관함 | 화면 | 카메라 썸네일 | 사진·영상 모아보기 |
| 뷰어 | 화면 | 보관함 항목 탭 | 단일 항목 감상·관리 |
| 설정 | 화면 | 카메라 상단 | 기본값·정보 (최소) |
| 필터 언락 | 모달 시트 | 잠긴 필터 탭 | 라이프타임 구매·복원 |
| 공유 시트 | 모달 시트 | 뷰어 공유 | 시스템 공유 |

## 3. 화면별 상세

### 3.1 권한 프라이밍 (첫 실행)
- iOS 시스템 팝업 **전에** 따뜻한 톤으로 "왜 필요한지" 1화면 설명 → 그다음 실제 권한 요청.
- 권한: 카메라(필수), 마이크(영상), 사진 라이브러리(저장/내보내기).
- 거부해도 앱은 동작(안내 화면 + 설정 유도). **계정 생성·이메일 입력 없음.**

### 3.2 카메라 · 홈 (허브)
- Phase 1 스펙 그대로: 상단 바(플래시·비율·설정) / 라이브 프리뷰 / 하단(필름통 피커·셔터·전환·보관함 썸네일).
- 잠긴(라이프타임) 필터 탭 → **필터 언락 시트**.
- 좌하단 썸네일 탭 → **보관함**.

### 3.3 보관함
- 사진·영상 통합 그리드 (촬영 날짜 기준).
- 항목 탭 → **뷰어**. 뒤로 → 카메라.
- 비파괴 구조라 썸네일은 *필터 적용된* 미리보기.
- 빈 상태: 따뜻한 일러스트 + "첫 장면을 담아보세요".

### 3.4 뷰어
- 단일 항목 풀스크린, 좌우 스와이프로 이전/다음.
- 액션: **공유**, 삭제, **필터 교체**(비파괴!), 날짜 스탬프 토글, 정보(촬영시각·비율).
- 공유 → **공유 시트**.

### 3.5 설정 (최소)
- 기본 비율, 날짜 스탬프 기본값, 햅틱 on/off, 그레인 강도(후속).
- 정보: 버전, 오픈소스 라이선스, 문의.
- **라이프타임 구매 복원**(Restore).
- ❌ 계정/프로필/로그인 항목 자체가 없음.

### 3.6 필터 언락 (모달 시트)
- 잠긴 필터 미리보기 + 라이프타임 1회 구매 + 복원.
- 핵심 카메라·저장은 막지 않음(무료). 무료/유료 구분 명확.

### 3.7 공유 시트 (모달)
- iOS 표준 공유(`UIActivityViewController`). 익스포트는 비파괴 렌더 결과.
