# Mellow Phase 2 — 영상 스펙 v0.1 (초안)

작성일: 2026-07-22
상태: **초안** — 미결 항목 존재, 구현 착수 전 §10 확인 필수

## 1. 범위와 단계

Phase 2는 두 단계로 분리한다. **2a만으로 완결된 제품**이며, 2b는 2a가 디바이스에서 검증된 뒤 설계한다.

| 단계 | 범위 |
|---|---|
| **2a** | 영상 촬영 → 트림 → 단일 클립 저장·재생·익스포트 |
| **2b** | 3장 스택 합성 + 텍스트 레이어 |

2b는 규모상 Phase 1에 준하는 작업이다. 2a의 열·용량 실측 결과가 2b 설계의 입력이 된다.

## 2. 촬영

### 2.1 녹화 길이
- 기본 녹화 단위: **최대 10초**
- 사용자는 넉넉히 찍고 트림 단계에서 2~3초를 선별한다 (§3)

### 2.2 필터 고정 — 확정
**필터는 녹화 시작 시 고정되며, 녹화 중에도 촬영 후에도 변경할 수 없다.**

- 근거: 브이로그 테이크의 무결성. "필름 한 롤을 넣고 찍는다"는 제품 문법
- 크로스페이드 전환 효과는 어느 모드에도 불필요 (`FrameProcessor.crossfade` 삭제 완료, 커밋 `abf4a4f`)
- 스틸의 즉시 스왑(L3 Decision B)은 그대로 유지 — 이 조항은 영상에만 적용

**주의:** "사용자가 필터를 바꿀 수 없다"는 것과 "LUT 렌더를 수정할 수 없다"는 별개다. §4의 저장 모델은 개발자가 LUT 자체를 개선할 여지를 남긴다.

### 2.3 해상도 상한 — 확정
**v1 상한: 짧은 변 720px 고정**

촬영 비율은 **§10-1에서 2.4:1 단일 고정으로 확정** — 저장 해상도는 **1728 × 720** 한 종이다.
(아래 근거 문단은 짧은 변 720 상한을 결정할 당시의 다중 비율 검토 기록.)

**"짧은 변 고정"을 택한 근거:**
- 세로 앱에서 클립은 화면 폭에 맞춰 배치되므로, 체감 선명도를 결정하는 것은 **폭**이다. 폭을 720으로 고정하면 비율을 바꿔도 선명도가 균일하다
- 긴 변 고정 시 1:1은 거의 네이티브(1.03배), 2:1은 2.06배로 업스케일되어 **같은 앱에서 비율만 바꿔도 화질이 달라 보인다**
- 최대 픽셀이 1.04M로 억제된다 (긴 변 고정은 1:1이 1.64M)

**업스케일 실측치** (전폭 재생 시 화면 폭 ÷ 720):

| 기기 | 화면 폭(px) | 업스케일 |
|---|---|---|
| 13 mini | 1125 | 1.56× |
| iPhone 12 | 1170 | 1.63× |
| 17 Pro Max | 1320 | 1.83× |

전 라인업 1.56~1.83배, 편차 17%. 기기 간 차이가 작아 단일 해상도 선택이 일관되게 동작한다.

**캡처 요구사항:** 트림본 1728×720(§10-1)을 업스케일 없이 얻으려면 소스 폭 ≥ 1728이 필요 → 세션은 최소 **1920×1080**으로 받아 크롭한다. (기존 최솟값 1280×960은 §10-1 확정 전 다중 비율 검토 기준 — 2.4:1 크롭 시 1280×533으로 목표 미달.)

### 2.4 LUT 준비 게이트 (Phase 2 설계 입력)
녹화 시작 시점은 LUT 로드 완료를 요구할 **자연스러운 게이트**다.

- 스틸에서는 프리페치 패스스루 창이 `.lutbin` 이후 ~0.1ms라 도달이 사실상 불가능하다
- 영상은 노출 시간이 길어 클립 앞부분이 통째로 어긋날 수 있다
- 선택된 slug의 LUT가 로드되지 않았으면 녹화 버튼을 비활성화한다 — "필름이 안 들어간 카메라는 녹화되지 않는다"

이 결정은 스틸의 WYSIWYG 프리페치 갭(백로그 기록됨)도 같은 논리로 닫을 수 있다.

## 3. 트림

- 10초 원본에서 사용자가 구간을 선택한다 (권장 2~3초, 강제 아님)
- 트림은 **구간 선택**이며, 원본은 트림 확정 시점까지 보존된다 (§4.2)
- UI: 스크러버 + in/out 포인트 + 프리뷰 재생

**미결:** 트림하지 않고 편집화면을 벗어난 경우의 처리 (10초 전체 보존 / 클립 폐기 / 확인 다이얼로그)

## 4. 저장 모델과 원본 수명주기

### 4.1 3계층 구조 — 확정

```
원본 (10초, 무필터)      → 트림 완료 시 삭제 (§4.2)
  ↓
트림본 (2~3초, 무필터) + filterID   ← 진실의 원천, 영구 보존
  ↓
구운 재생본 (LUT 적용)              ← 파생 캐시, 언제든 재생성
```

**이 구조를 택한 근거:**
- 사진 파이프라인과 동일한 패턴이다 (원본 + filterID → `CaptureThumbnailRenderer` 렌더). 새 개념이 없다
- 갤러리 스크롤 시 재생본을 읽으므로 3장 동시 재생에서도 LUT 렌더 부하가 0이다
- **LUT를 개선하면 캐시만 버리고 다시 구워 과거 클립까지 개선된다.** §6의 필터 세기 검증 결과가 나중에 나오더라도 되돌릴 수 있다
- 용량 부담이 작다: 원본을 지우므로 2~3초 트림본 추가는 미미하다

### 4.2 원본 삭제 조건 — 확정

**삭제 조건 = 트림본 쓰기 성공 확인 + 사용자의 명시적 완료**

"편집화면 이탈"이 아니라 **"완료"**다. 이유:

- "이탈"은 의도적 행위(완료·뒤로가기)와 비의도적 종료(전화 수신·메모리 부족 킬·크래시·배터리 방전)를 구분하지 못한다
- 이는 가정이 아니라 이 카테고리에서 실제로 발생한 실패 양식이다. Filmm은 앱 내 촬영 중 전화가 오면 녹화분이 삭제되어 사용자가 영상을 반복적으로 잃었고, 개발사가 1.2.2에서 iOS 기본 카메라처럼 저장되도록 수정했다
- Mellow는 여기에 "원본까지 삭제"가 얹히므로 더 파괴적이다

**요구사항:**
1. 트림본 쓰기 성공을 **확인한 뒤에만** 원본을 삭제한다
2. 비의도적 종료 시 원본이 남고, 다음 실행에서 복구 제안 ("편집 중이던 클립이 있습니다")
3. 사용자가 의도적으로 뒤로 나갈 때는 확인을 받는다 — 파괴적 동작에 대한 최소한의 예의

## 5. 합성 레이아웃 (2b)

### 5.1 스택 구조 — 확정
- 가로 **2.4:1** 클립 **3장**을 세로로 밀착 배치 (클립 비율은 §10-1에서 16:9 → 2.4:1로 확정 변경)
- 여백은 **위아래에만** 존재, 클립 사이에는 없음
- 배경은 **크림 `#F4EBDD`** — 순검정 금지 (디자인 시스템 §9)

### 5.2 기하학 검증
**[§10-1로 대체]** 아래 검증은 16:9 가정 당시의 기록. 확정 기하는 §10-1-2(2.4:1 슬롯, 캔버스 1080×1920) — 2.4:1은 클립 세로가 더 낮아(폭의 1/2.4) 3장 소요가 1.25배로 줄어 여유가 더 커진다.

클립을 전폭으로 놓으면 각 클립의 세로 = 폭 ÷ 비율. 화면 가용 세로는 폭의 2.17배(19.5:9).

| 구성 | 소요 세로 | 결과 |
|---|---|---|
| 16:9 가로 × 3 | 1.69배 | ✅ 0.48배 여유 |
| 1:1 × 2 | 2.0배 | 빠듯 |
| 9:16 세로 × 2 | 3.56배 | ❌ 초과 |

**기기별 여백:**

| 기기 | 클립 3장 | 여백(위/아래 각) |
|---|---|---|
| 13 mini (375×812pt) | 633pt | 90pt |
| 17 Pro Max (440×956pt) | 742.5pt | 107pt |

**세이프 에어리어가 여백 안에 들어온다.** 상단 인셋 50~62pt, 하단 34pt < 여백 90~107pt. 전 라인업에서 클립이 다이나믹 아일랜드·홈 인디케이터에 닿지 않으며, 기기별 예외 처리가 불필요하다.

### 5.3 재생 모드 — 확정
**동시 재생과 순차 재생을 모두 지원한다.**

성능 예산은 **최악(동시) 기준**으로 산정한다 (§8).

## 6. 텍스트 레이어 (2b)

### 6.1 데이터 모델
텍스트는 영상에 굽지 않고 **메타데이터로 저장, 렌더 시점에 합성**한다. 기존 비파괴 원칙의 확장.

텍스트 항목 하나가 보유하는 필드:

```
clipIndex      어느 클립에 속하는가
x, y           정규화 좌표
startTime      표시 시작 (클립 기준 상대 시각)
endTime        표시 종료
content        내용
style          크기, 정렬, 그림자 on/off
```

**"원하는 타이밍에 원하는 곳에만"** — 세 클립 모두에 나타나지 않으며, 각 텍스트가 독립적인 위치와 시간 구간을 갖는다.

이는 단순 메타데이터가 아니라 **텍스트 레이어 + 타임라인**이다. 편집 UI(탭 추가, 드래그 이동, 타이밍 조절)와 익스포트 합성(`AVMutableVideoComposition` + `CALayer`)이 모두 필요하다.

### 6.2 자동 채움
- 날짜·시간: `Capture`에 이미 존재 → 자동 채움 후 사용자 수정 가능
- 장소: 신규 필드, **수동 입력으로 시작** (GPS 자동 취득은 권한 증가를 수반하므로 별도 결정)

### 6.3 서체 — 명조체
톤앤매너 근거: 산세리프는 "앱", 명조는 "기록". 필름 다이어리에 부합.

**용량 문제:** 한글 완성형 11,172자로 서체 하나가 통상 5~20MB. 사용자가 장소명을 직접 입력하므로 **서브셋팅이 불가능**하다.

| 후보 | 장점 | 확인 필요 |
|---|---|---|
| **AppleMyungjo** (시스템) | 번들 0바이트, 라이선스 무관 | 무게 1종, 렌더 품질 |
| 나눔명조 | 따뜻함, OFL | 무게 3종, 용량 |
| 본명조 / Noto Serif KR | 커버리지 최상 | 용량 최대 |

**AppleMyungjo부터 실기 확인.** 사용 가능하면 용량 문제가 소멸한다. 라이선스는 OFL이라도 번들 조건을 직접 확인해야 한다 (LUT 자체 저작 원칙과 동일한 성격의 결정).

### 6.4 가독성
텍스트가 밝은 배경(흰 침구 등) 위에 놓이면 읽히지 않는다. 영상은 프레임마다 배경이 바뀌므로 안전한 위치가 없다.

- **그림자는 사용자 선택**으로 제공
- **기본값 켜짐 권장** — 대부분의 사용자는 기본값을 바꾸지 않으므로, 기본값이 곧 실질적 디자인 결정이다
- 그림자 색은 **들린 블랙 `#3B362E`** 저불투명 + 큰 블러. 순검정 금지
- 글자색은 순백 대신 **크림 `#F4EBDD`**

**실물 검증 필요:** 목업에서 적절해 보여도 움직이는 영상 위에서 다를 수 있다.

## 7. 익스포트

### 7.1 캔버스 — 확정
**9:16** — 비율 결정 유지(§10-1 INV-3). 해상도·배치 수치는 **§10-1-2로 대체**: 캔버스 **1080×1920**, 슬롯 1080×450 ×3(y = 286/736/1186), 크림 띠 상 286 / 하 284.

(아래 후보 비교는 §10-1 확정 전 720 기준 검토 기록 — 비율 선택 근거로만 유효.)

**후보 비교:**

| 방식 | 크기 | 문제 |
|---|---|---|
| 딱 맞게 (16:27) | 720×1215 | 재생기가 **검정으로 채움** → §9가 공유물에서 붕괴 |
| **9:16** | 720×1280 | 앱에서 본 여백(11%)보다 파일의 띠(2.5%)가 훨씬 얇음 |
| 화면 비율 | 720×1562 | 비표준 비율, 공유 시 어색 |

9:16을 택한 근거: **크림 여백이 파일에 포함되면서 동시에 표준 규격**인 유일한 선택지. 릴스·쇼츠 호환.

**알고 가는 대가:** 앱 화면과 익스포트 파일의 여백 비율이 다르다 (11% vs 2.5%). 동일 구성이 파일에서 더 빡빡해 보인다.

## 8. 성능 예산과 측정 게이트

### 8.1 부하 구조
굽기 캐시(§4.1)로 인해 **재생 시 LUT 렌더가 없다.** 부담은 디코드에 집중된다.

| 항목 | 단일 클립 | 스택 3장 동시 |
|---|---|---|
| 디코드 | 1스트림 720×1280 (0.92M px/frame) | **3스트림 1280×720 (2.76M px/frame)** |
| 합성 | — | 720×1215 (0.87M px/frame) |
| LUT 렌더 | 0 (캐시) | 0 (캐시) |

디코드가 3배이며 이것이 지배적 비용이다. 합성 결과 자체는 단일 클립과 비슷하다.

**[§10-1 반영 재산정 필요]** 위 표는 §10-1 확정 전 수치. 트림본 1728×720 기준 동시 3스트림 = **3.73M px/frame**(표의 1.35배), 합성 캔버스 1080×1920 = 2.07M px/frame. §8.2 측정 게이트에서 실측으로 확정할 것.

### 8.2 측정 게이트 — 2b 진입 전 필수
Session 9에서 스틸 파이프라인 베이스라인을 확립했듯, **동시 3장 재생을 Instruments로 재측정한 뒤** 2b 설계를 확정한다.

**측정 없이 3장 동시 재생을 전제하지 않는다.** 결과에 따른 대응 선택지:
- (a) 그대로 진행
- (b) 스택 모드에서만 해상도 하향 (예: 짧은 변 540)
- (c) 동시 재생을 2장으로 제한

## 10-1. 촬영 비율과 합성 레이아웃

### 10-1-0. 결정 요약
- 레이아웃: 3컷 세로 스택 1종 영구 고정. 확장 계획 없음.
- 레이아웃 선택 UI 없음. 사용자 노출 레이아웃 명칭 없음.
- 촬영 비율: 2.4:1 단일 고정. 레이아웃에서 파생된 값이며 독립 선택지가 아님.

### 10-1-1. 불변식
- INV-1: 촬영 비율 ≡ 슬롯 비율. 합성 단계는 배치 + 등방 스케일만 수행,
  크롭 결정 없음. 크롭은 트림본 기록 시점 1회만 발생.
- INV-2: 슬롯 렌더 폭 <= 소스 클립 폭. 업스케일 금지.
- INV-3: 익스포트 캔버스는 항상 9:16.
- INV-4: 저장 진실 = 무필터 트림본 + filterID + slotIndex + compositionVersion.
  합성 결과물은 전량 캐시.
- compositionVersion: Int 상수, 현재 값 1. 여백/비율 수치 조정 시 과거 영상
  재합성 호환 보장용. layoutID는 두지 않음 (형제 부재).

### 10-1-2. 기하
캔버스 1080x1920 (9:16)
슬롯 1080x450 (2.4:1)
좌우 여백 0, 슬롯 간 갭 0
y 오프셋 286 / 736 / 1186
상단 크림 286, 하단 크림 284
검산: 286 + (450*3) + 284 = 1920
트림본 저장 1728x720 (2.4:1, 짧은 변 720 상한)
합성 스케일 0.625 등방 (1080/1728 = 450/720), 축소이므로 INV-2 충족

상하 비대칭(286/284)은 의도된 값. 균등 285/285는 y 오프셋을 전부 홀수로
만들어 YUV 4:2:0 크로마 평면과 어긋나며, 크림(#F4EBDD)과 리프티드
블랙(#3B362E)의 높은 대비로 슬롯 경계 1px 색번짐 발생 가능. 2px 조정으로
전 오프셋 짝수화. 0.1% 차이는 육안 감지 불가.

### 10-1-3. 구현 제약
- 컴포지터는 y = 286 / 736 / 1186 정수 좌표로만 렌더. 부동소수점 스케일 금지.
- 트림본은 정확히 2.4:1로 크롭 기록. 1px 오차 시 슬롯 경계 크림 헤어라인 노출.
- 슬롯 비율 homogeneous 고정. 혼합 비율 레이아웃은 v1 범위 밖
  (세션당 뷰파인더 마스크 1개 원칙 위배).

### 10-1-4. 노출 연속성
갭 0이므로 클립 간 노출 차가 경계에서 명도 계단으로 드러남. 필터 완전 고정으로
앱 측 보정 수단 없음. 노출 연속성은 사용자 촬영 책임으로 정의.
완화책: 촬영 UI에서 기촬영 슬롯 썸네일을 뷰파인더 옆에 상시 노출.

### 10-1-5. 10-2 이월
- 미완성 슬롯 처리 (3슬롯 중 2장만 촬영된 경우)
- 동시 재생 vs 순차 재생, 총 길이 = max(클립 길이) 시 짧은 클립 잔여 구간 처리

## 11. 영상 필터 세기 검증 (2a 필수 항목)

**문제:** 사진에서 적절한 LUT가 영상에서 과할 수 있다.

- 사진은 한 장을 응시하므로 강한 색이 "분위기"로 읽힌다
- 영상은 계속 흐르므로 눈이 색 변화를 지속 감지하여, 강한 그레이딩이 "장면"이 아니라 "필터"로 보이기 시작한다

**위험군:**
- 채도가 센 필터(`color`)는 움직임 위에서 인공적으로 보인다
- 콘트라스트가 센 필터는 저조도 노이즈를 증폭한다. 영상은 프레임마다 노출이 흔들려 더 두드러진다
- 컬러 시프트가 강하면 화이트밸런스 자동 조정 시 색이 출렁인다

**검증 방법:** 2a 프로토타입에서 9종 전부로 동일 장면을 촬영해 육안 판정.

**결과에 따른 갈래:**
- (a) 그대로 사용
- (b) 영상용 저강도 변형 LUT 제작
- (c) 일부 필터를 영상 모드에서 제외

§4.1의 굽기 캐시 구조가 이 검증을 **나중에 해도 안전하게** 만든다 — LUT를 고쳐도 과거 클립이 함께 개선된다.

**참고 (Original의 지위):** 샤디(Shardee)는 "편집 없음, 필터 없음"을 정체성으로 삼아 카테고리 상위권에 있다. 무필터 결과물도 충분히 담백하다는 시장 증거이며, 영상 모드에서 **Original은 "필터를 안 고른 상태"가 아니라 떳떳한 선택지**로 대접받아야 한다.
