# Mellow — UI & 인터랙션 가이드 v0.1

> **역할.** 디자인 시스템(시각 언어)과 화면 스펙(개별 화면) 사이를 잇는다. 모든 화면이 재사용할 **컴포넌트·모션·햅틱·상태** 규칙.
> **원칙 상속.** 따뜻한 페이드 · 들린 블랙 · 덜어내기 · 느긋한 모션. 연결: 디자인 시스템 v0.2.

---

## 1. 컴포넌트 라이브러리

> 공통: 모서리 카드 12px·컨트롤 8px, 보더 0.5px(`mellowBorder`), 그림자는 들린 블랙. 폰트 = 본문 산세리프, 포인트 세리프.

### 1.1 버튼
| 종류 | 모습 | 용도 |
| --- | --- | --- |
| Primary | 앰버(`#C97F3E`) 필, 크림 텍스트 | 핵심 액션(구매·확인) |
| Secondary | 페이퍼 배경 + 0.5px 보더 + 잉크 텍스트 | 보조 액션 |
| Ghost/아이콘 | 배경 없음, 잉크 아이콘 | 상단 바·툴바 |
| Destructive | 텍스트만 빨강 계열(절제) | 삭제 |
- 상태: default / pressed(살짝 어둡게 + 스케일 0.97) / disabled(40% 불투명). 최소 터치 44×44pt.

### 1.2 셔터 버튼 (특수)
- 큰 원형, 앰버 링 + 크림 중앙. 누르면 중앙이 살짝 수축 + 햅틱.

### 1.3 필름통 필터 칩
- 시그니처색 사각 + 이름. 선택 = 앰버 링 + 살짝 떠오름.
- 잠김 = 우상단 자물쇠 글리프 + 살짝 디머.

### 1.4 슬라이더 (노출·강도)
- 트랙 = 라이트탄, 채움 = 앰버, 핸들 = 크림 + 0.5px 보더.
- 값 라벨은 드래그 중에만 표시.

### 1.5 토글 / 세그먼트
- 토글 on = 앰버, off = 라이트탄. 세그먼트(비율 등) 선택 = 페이퍼 위 떠오른 칩.

### 1.6 토스트 / 인라인 알림
- 하단 페이퍼 카드 + 잉크 텍스트, 2.5초 자동 소멸. 에러는 빨강 텍스트 액센트(배경은 그대로 따뜻하게).
- ❌ 시스템 회색 alert 남발 금지.

### 1.7 모달 시트
- 하단에서 올라오는 시트, 상단 그래버 핸들, 페이퍼 배경, 둥근 상단 16px.
- 배경 딤 = 들린 블랙 45% (순수 검정 ❌).

### 1.8 빈/로딩 상태 카드
- 따뜻한 일러스트 + 한 줄 카피. 로딩은 부드러운 페이드/필름 감김 모션(스피너 지양).

## 2. 모션 원칙

| 상황 | 시간 | 이징 |
| --- | --- | --- |
| 필터 전환 크로스페이드 | ~180ms | ease-in-out |
| 화면 전환(푸시) | ~300ms | 표준 ease |
| 시트 등장/퇴장 | ~280ms | spring(부드럽게) |
| 버튼 프레스 | ~100ms | ease-out |
| 토스트 인/아웃 | ~200ms | ease |

- **원칙:** 느긋하고 부드럽게. 빠르고 튀는 모션 ❌. 동시에 여러 요소가 제각각 움직이지 않게.
- `prefers-reduced-motion` 존중 — 모션 최소화 모드 제공.

## 4. 아이콘 & 일러스트

- 아이콘: 얇고 둥근 라인, 일관된 굵기. 채움 아이콘 지양.
- 색: 기본 잉크, 비활성 카라멜, 강조 앰버.
- 일러스트(빈 상태 등): 따뜻한 팔레트 + 약간의 종이 질감. 과한 디테일 ❌.

---

## 5. 공통 상태 (모든 화면 공유)

각 화면은 아래 상태를 정의해야 한다:
- **빈 상태(Empty)** — 따뜻한 안내 + 다음 행동 유도.
- **로딩(Loading)** — 부드러운 페이드, 콘텐츠 자리 유지.
- **에러(Error)** — 따뜻한 토스트/카드 + 재시도. 차가운 시스템 경고 톤 ❌.
- **권한(Permission)** — 거부 시 설명 + "설정 열기". 앱 비충돌.
- **오프라인** — 로컬 앱이라 대부분 무관(구매·복원만 네트워크 필요 → 그때만 안내).

## 6. 접근성 & 대비 ⚠️

> 우리의 따뜻한 저대비 팔레트는 예뻐서 위험하다. 미감을 지키되 가독성을 깨지 않도록 규칙을 둔다.

- **본문 텍스트는 대비 확보.** 보조 텍스트색 카라멜(`#B0855C`)을 페이퍼 위 **장식·라벨에만** 쓰고, 실제 읽어야 하는 본문엔 잉크(`#4A443A`) 또는 더 진한 전용 토큰을 사용.
- **고대비 텍스트 토큰 보강 제안:** `mellowTextStrong`(예: `#3A352D`/들린 블랙 근처)을 추가해 작은 본문·캡션에 사용 → WCAG AA(4.5:1) 목표.
- **버튼·인터랙티브**: 텍스트와 배경 대비 충분히. 앰버 필 위 텍스트는 크림(밝음)으로.
- **상태를 색에만 의존 금지** — 잠금/선택/에러는 아이콘·형태도 함께.
- **동적 타입(Dynamic Type)** 지원, 최소 폰트 11pt 이상.
- **VoiceOver** 라벨: 아이콘 버튼에 `accessibilityLabel`. 셔터·필터 칩 등 커스텀 컨트롤 명시.
- **모션 최소화** 모드(§2) 제공.
